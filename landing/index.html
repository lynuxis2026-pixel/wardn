SCAN

Explainable trust scores

Every signal carries a human-readable reason: broad filesystem, shell launcher, unpinned package, remote transport, credentials in env. No black box.

SANDBOX

Policy-first isolation

Per-server filesystem whitelist, network on/off, env-whitelist. Out-of-policy tool-calls are rejected before the MCP server ever sees them. Docker adds container isolation on top when available.

GATEWAY

Byte-faithful proxy

Sits between any MCP client and any stdio server. Logs every JSON-RPC call with method, id, and duration. Streams live to the dashboard via SSE.

DASHBOARD

Lynuxis-styled console

Open http://127.0.0.1:7331. Server list with trust badges, signals, sandbox toggle, live tool-call log. Served by the gateway daemon.

REWRITE

One-command routing

Rewrites Claude Desktop / Cursor / VS Code configs to route every MCP server through wardn. Backed up byte-for-byte. wardn rewrite restore undoes it.

REGISTRY

Community trust layer

Curated reputation data per MCP package: verified publisher, popularity, known-bad flags. Pull requests welcome — every entry is one short YAML form.

Quickstart

  1. npx wardn scan

    Lists every MCP server in your installed clients and scores each one. Exits non-zero if anything risky is found — drop it into CI.

  2. npx wardn sandbox enable filesystem --path ~/safe

    Locks the filesystem server to a single directory. Network off by default. Re-scan shows it as TRUSTED ⛨ sandboxed.

  3. npx wardn gateway start

    Starts the local daemon. Dashboard at http://127.0.0.1:7331, SSE log at /api/events.

  4. npx wardn rewrite apply

    Routes existing client configs through the gateway. Restart the client and every tool-call now flows through wardn.

  5. npx wardn demo

    Spawns the bundled evil-MCP and proves the sandbox blocks four real-world attack vectors before they reach the server.
