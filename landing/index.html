What wardn does

SCAN

Explainable trust scores

Every signal carries a human-readable reason. No black box. No vendor magic.

RISKYbroad-fs  — "/" filesystem access
RISKYshell-exec  — bash / sh launcher
REVIEWfloating-version  — unpinned community package
REVIEWremote-transport  — data leaves your machine
INFOverified  — Anthropic / Microsoft / Notion / …
INFOsandboxed ⛨  — policy is active for this server
GATEWAY

Byte-faithful proxy

Sits between client and MCP server. Logs every JSON-RPC call.

OUTtools/list
IN(response) · 11ms
OUTtools/call read_secret
SYSblocked by policy
SANDBOX

Policy-first isolation

Per-server filesystem whitelist, network on/off, env whitelist. Out-of-policy tools/call rejected before it reaches the server. Optional Docker / bubblewrap on top.

DASHBOARD

Lynuxis-styled console

Open 127.0.0.1:7331. Trust badges, signals, sandbox toggle, live tool-call log streamed over SSE. Served by the gateway daemon.

REWRITE

One-command routing

Rewrite Claude Desktop / Cursor / VS Code configs to route through wardn. Backed up byte-for-byte. rewrite restore undoes everything.

REGISTRY

Community trust layer

Curated reputation per MCP package — verified publishers, popularity, knownBad. Pull requests welcome; every entry is one short YAML form.

Quickstart

  1. wardn scan

    Lists every MCP server in your installed clients and scores each one. Exits non-zero if anything risky is found — drop it into CI.

  2. wardn sandbox enable filesystem --path ~/safe

    Locks the filesystem server to a single directory. Network off by default. Re-scan now shows TRUSTED ⛨ sandboxed.

  3. wardn gateway start

    Starts the local daemon. Dashboard at 127.0.0.1:7331, SSE log at /api/events.

  4. wardn rewrite apply

    Routes existing client configs through the gateway. Restart the client and every tool-call now flows through wardn.

  5. wardn demo

    Spawns the bundled evil-MCP and proves the sandbox blocks four real-world attack vectors before they reach the server.

Share your trust state

The gateway exposes a live SVG badge at /api/badge.svg. Drop it into any README and your readers see the trust state of your MCP stack in one glance.
